【Vibe Coding】地球 World tour

By YH Chen 30 December 2025

這篇實作是參考，李慕約的 365 AI 訂閱計劃中的 案例 #122 地球

地球視覺化的原始範例，來自 d3.js Gallery 中的 World tour

可以透過 GitHub Pages 連結，造訪 Vibe Coding 後的成果

歡迎點進去 GitHub Pages 和 World tour，比較看看 Vibe Coding 前後的差異

接著，介紹一下 Vibe Coding 的步驟以及 Vibe Coding 時的思考脈絡 (Vibe Coding 工具：OpenAI Codex )

  1. 把專案部署到 GitHub Pages

  2. 新增 UI（路線查詢介面）

    • 新增下拉式選單：使用者自行選擇【出發地】/【到達地】

    • 新增【查詢】按鈕：只有按下查詢才會送出本次路線並更新「目前路線顯示」

    • 細節調整：

      • 成功查詢後，上一段的【到達地】自動預設為，下一次查詢的【出發地】，但【出發地】仍保留可手動修改的彈性
      • 成功查詢後，【到達地】清空為未選狀態，提供使用者自行選擇
      • 再次按下【查詢】前，變更下拉選單的【出發地】/【到達地】，都不會直接影響「目前路線顯示」；畫面會維持在上一筆成功查詢的結果

  3. 網頁畫面調整

    • 國家名稱顯示改為「中文 + 英文」
    • 網頁上不要顯示程式碼
    • 路線視覺與動畫調整：使用「箭頭標記」呈現移動方向，讓路徑具有方向性；到達後路線需消失，不要一直停留在畫面上

  4. 增加互動功能（滑鼠拖拉旋轉地球）

    • 動畫結束後，使用者可自由拖拉旋轉地球，且視角保持使用者操作結果（不會自動重新對準【到達地】）
    • 動畫結束後【出發地 = 紅色標示】/【到達地 = 綠色標示】仍需持續顯示，方便使用者拖拉找回【出發地】與【到達地】
