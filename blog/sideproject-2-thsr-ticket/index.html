【Vibe Coding】自動化高鐵查票工具

By YH Chen 12 June 2026

AI Vibe Coding ，開發了一款 Google Apps Script (GAS) 自動化高鐵查票工具

可以透過連結，造訪成品

使用情境

相信大家都有忘記訂高鐵票或臨時需要買票的經驗，這時候就只能趕快拿起手機瘋狂刷票

但是，要在短時間內反覆手動刷新網頁和 App，真的是超級煩！

我本來想說可以試試看用 ChatGPT 網頁版的代理模式

讓它自動幫忙搜尋有沒有釋票，但是它會一直卡關在輸入驗證碼這個地方

所以就換個方式用 Vibe Coding 開發出自動化查票工具

Vibe Coding 步驟

Step 1. 取得 TDX API

先到 TDX 註冊一個帳號，取得 API ( Client ID / Client Secret )

Step 2. 定義問題和釐清需求

先整理 Prompt，而不是直接叫 AI 開始寫程式 ，

可以把 Prompt 拆成至少兩個部分：遇到的問題(現況)、設定驗收條件(期待)

### 遇到的問題(現況)

有時候我會忘記訂高鐵車票，會想要透過自動化搜尋還有沒有票可以買，
但透過自動化搜尋會碰到查詢高鐵訂位時需要輸入驗證碼，通常電腦在執行輸入驗證碼這一步都會出錯，
這樣我就沒辦法自動化搜尋，而是要自己手動搜尋 。
  
### 設定驗收條件 (期待)

- 使用者可以自行輸入特定日期、時段、起始站、抵達站
- 查詢結果要包含班次、起始站、發車時間、抵達站、抵達時間、票況
- 特定日期與時段有票時，系統會寄 Email 通知
- 沒有票時不需要寄信

初版出來後，我會反覆進行，優化使用者操作體驗、安全性優化 這兩個步驟，實際調整的內容如下

Step 3. 優化使用者操作體驗

  • 加入 onOpen()，在 Google Sheet 頁面建立自訂選單，讓使用者不需要進入 Google Apps Script 後台也能執行功能

  • 支援單次查詢或定期自動化查詢

  • 加入「最新的執行時間」欄位，方便確認自動化查詢是否仍在執行

  • 使用者可以直接在 Google Sheet 上方選單輸入或更新自己的 API。整個過程不需要進入 Apps Script 後台，降低操作門檻

  • 拿掉 「避免重複寄信」功能， 改成只要每次查詢有符合條件的班次就寄出 Email，讓通知邏輯更直覺，降低錯過票況變化的風險

Step 4. 安全性優化

  • TDX API 不直接寫死在 Google Apps Script 程式碼中，而是讓使用者自行輸入，避免個人 API 外洩

  • 加入停止自動查詢功能，避免造成不必要的 API 呼叫或 Email 通知

專案成果

這個工具結合了 Google Apps Script (GAS) 零成本部署的優勢

成功將原本需要人工重複刷票的繁瑣過程，轉換成可以自動化定時查詢，並在有票時主動寄送 Email 通知的查票系統

成果心得

在這次實作過程中，我感受到現在真正困難的地方，已經不再是寫程式本身了，而是有沒有能清楚定義問題、拆解需求的能力

也就是說，現在真正重要的是如何「問對問題」，並且透過 Vibe Coding 的方式快速驗證迭代，把想法落地貼近真實的使用情境
